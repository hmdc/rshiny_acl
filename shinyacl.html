shinyacl package

shinyacl.ShinyACL

The ShinyACL module provides functionality to add and remove users from .shiny.conf files inside Shiny application directires. The ShinyACL module also handles application restarts via restart.txt when ACLs for an application has changed

class shinyacl.ShinyACL.ShinyACL(__root__='/nfs/home/E/esarmien/shared_space')[source]

The ShinyACL class provides methods which add, remove users to an application’s ACL in .shiny.conf and restarts that application.

__write__(app, authstring)[source]
Writes the .shiny_app.conf file inside the app directory by
modifying the required_user line. No need to use this as higher level methods like add_user call this function with the proper arguments.
Parameters:
  • app (str) – Fully qualified path to application directory
  • authstring (str) – The authentication line to write
Example: 
>>> from shinyacl import ShinyACL
>>> ShinyACL().write('/nfs/www/shinyserver/vpal/hello',
      'required_user esarmien@g.harvard.edu;')

get_users(app)[source]

Returns a list of users which are allowed to access an application. Raises exception if the app argument is a path which is not a RShiny application. Returns an empty array if no users or configured or if .shiny_app.conf does not exist in the app directory.

Parameters:app (str) – Fully qualified path to application directory
Return type:list
Raises:shinyacl.ShinyACLExceptions.ShinyACLNotAShinyApp
Example: 
>>> from shinyacl import ShinyACL
>>> ShinyACL().get_users('/nfs/www/shinyserver/vpal/hello')
['dtingley@g.harvard.edu', 'v@v.com']
